🤔 关于「进度」的思考

数值百分比（如 40%、85%）对"边做边想"的项目没有意义——因为没人知道"100%"长什么样。我建议替换方案：


阶段制（5段），替代百分比：

① 需求/构思 → ② 搭建框架 → ③ 核心功能 → ④ 打磨完善 → ⑤ 上线/移交

每个阶段没有"完成度"细分——要么到了，要么没到。比"60%"更诚实。


完成的判定标准：{需求明确 + 交付物具体} → 做完了就是做完，不需要百分比。


已完成项目存哪？ 每个 Agent 组下面可以加一行灰色"已完成项目 N 个 →"的折叠入口，点开看。或者统一放在页面底部的"📦 已完成/已归档"区。我倾向于后者——清楚、不分散注意力。
