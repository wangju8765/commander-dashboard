### 技术教训
1. **先出示意图再动手** — 这次的项目详情页改造是先画了 mockup 给用户看，确认后才写代码，省去了反复修改。这是正确流程。
2. **CSS 层就要把两套视图互相隐藏** — 之前只靠 JS 的 switchView() 来隐藏另一个视图，JS 执行前两套视图同时可见。修复：手机媒体查询里加 `#desktopView{display:none}`，桌面媒体查询里加 `#mobileView{display:none}`
3. **iOS scrollTop 需要在 DOM 渲染后才设** — `-webkit-overflow-scrolling: touch` 有已知 bug，惯性滚动后 `scrollTop=0` 不生效。修复：先渲染内容再归零 + `scrollTo(0,0)` 双保险 + setTimeout 16ms 补刀
4. **Service role key 可能没权限** — Supabase 的 service_role key 报 permission denied，但 anon key 可以 SELECT + UPDATE（因为 RLS allow all）。需要检查 service_role 的权限设置。
5. **不要因为一个 bug 回滚全部功能** — 用户报告的字小问题只是个别 bug，我却回滚了 v0.7.1-v0.7.4 所有功能，用户很生气。应该针对性修问题，不要动其他功能。
6. **列表和详情用同一个 description 字段导致列表卡片太长** — 详情页需要完整的 Markdown，列表只需要摘要。解决：列表用 `trunc(esc(p.description||''), 60)` 截断。
