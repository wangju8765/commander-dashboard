Claude Code 完全理解指南

生成日期：2026-05-28 · 面向零编程基础的你
🎯 一句话说清：Claude Code 是一个住在你终端里的 AI 程序员。 你告诉它"帮我实现用户登录"，它自己读项目、制定方案、写代码、 跑测试、迭代修复，直到任务完成。
🤖 它跟普通 AI 编程工具有什么不同？

这是一个本质区别，不是程度的差异。

💡 传统 AI 像自动补全——你打字，它猜你下一句。
Claude Code 像一个外包程序员——你给需求，它自己去调研、 写代码、测试、交付。
⚙️ 核心原理：Agent 工作循环

Claude Code 不是简单生成文字，而是一个持续循环的 Agent 系统：

你给任务
    ↓
┌─────────────────────────┐
│ 1. 理解需求              │
│ 2. 读代码库（扫描结构）   │
│ 3. 制定方案（要改哪些文件）│
│ 4. 执行（编辑+跑命令）     │
│ 5. 验证（跑测试）          │
│ 6. 迭代（失败就修）        │
└─────────────────────────┘
    ↓
任务完成，交付结果

这叫"工具循环"——每次决定用什么工具、 读什么文件、跑什么命令，结果反馈回来再做下步决策。 跟人类写代码完全一样：写完→检查→发现不对→改→再检查。

它能调用什么工具？
  • 读文件 —— 你不用手动打开相关文件
  • 编辑文件 —— 直接修改代码
  • 运行 shell 命令 —— npm test / git status / curl
  • 搜索代码 —— grep / find 定位相关内容
  • Git 操作 —— commit / push / 创建 PR
  • 浏览器 —— 打开网页测试 UI
  • MCP 工具 —— 连接数据库 / API / Slack 等外部服务
    • 💼 你能用它干什么？——8个实战场景
    🏗️ 场景1：接手新项目

    进入项目目录，启动 Claude Code。

    问："这个项目用什么技术栈？架构怎么设计的？"
    Claude Code 会自己翻文件，给你一份项目全景图。

    🐛 场景2：修 Bug

    把报错信息贴给它：

    我跑 npm run build 报了这个错：
Error: Cannot find module 'react-dom/client'
请帮我找到原因并修复

    它自动检查依赖、修复版本、重新构建验证。

    ✨ 场景3：写新功能
    帮我给用户模块加上"重置密码"功能
要求：发验证邮件 · 验证码有效期15分钟
      重置后发通知

    它分析现有结构，跨文件修改：路由、验证逻辑、邮件模板、 写单元测试、跑通后提交代码。

    🔄 场景4：代码重构

    "把所有 fetch 调用替换成 axios，保持相同的错误处理逻辑。"
    找所有相关文件，逐个替换，跑测试确认没改坏。

    🔍 场景5：代码审查

    "review 这次 git diff 中的改动"
    对照项目规范找潜在问题、性能隐患、安全漏洞。

    🚀 场景6：自动化部署
    帮我部署到生产环境
1. 运行测试  2. 构建  3. 部署到服务器
4. 验证成功  5. 失败就回滚
    🧪 场景7：写测试

    "给 auth 模块写单元测试，覆盖率 90%，跑测试，修补没通过的。"

    🔄 场景8：项目迁移

    "把这个 Python 项目迁移成 TypeScript，保持 API 接口不变。"

    🔧 进阶：定制你的 Claude Code
    📄 CLAUDE.md —— 项目级"记忆文件"

    在项目根目录放一个 CLAUDE.md，Claude Code 每次自动读。 像团队手册一样告诉它代码规范，生成的代码就符合你的风格。

    # 项目规范
- 使用 TypeScript 严格模式
- 测试覆盖率 > 80%
- API 必须有 try-catch 错误处理
    🧩 Skills —— 可复用的"技能包"

    预封装的工作流程指令，别人写好的、用完即走。

    \skills   # 查看可用技能
使用 sql skill 分析数据库表结构
使用 frontend-design skill 设计这个页面
    🔒 Hooks —— 自动化"守门员"

    特定事件触发时自动执行的脚本。和 CLAUDE.md 不同， Hooks 是强制执行的。比如修改文件后自动格式化、 执行危险操作前弹确认。

    🔗 MCP —— 连接外部世界

    MCP（Model Context Protocol）让 Claude Code 能连接外部服务： 数据库查询、调用 API、连接 Slack/Jira、查天气查股票。

    👥 Sub-agents —— 多 Agent 并行

    复杂任务分解给多个子 Agent。比如主 Agent 拆任务， Agent A 写后端、B 写前端、C 写测试，各自独立工作互不干扰。

    ⚠️ 它不是用来做什么的
  • 不是设计工具——不会帮你画 UI mockup
  • 不是产品经理——不会帮你定义需求
  • 不是非程序员工具——本质是为开发者设计的
  • 不能代替你思考——架构决策还得你来定
    • ❓ 常见问题
    我不会编程，能用吗？

    不推荐。它是给程序员用的效率工具。虽然它能写很多代码， 但你需要能读懂、评估、修正它的输出。

    跟 Cursor / Copilot 比呢？

    Claude Code 是 Agent，其他两个偏辅助补全。同一件事， Claude Code 能自己跑完整个流程。但 Cursor 的 IDE 体验更流畅。

    安全吗？贵吗？

    每次执行命令都经过权限审批。Pro \$20/月够个人用。 也可以用 API 按量付费。换国产模型成本更低。

    国内网络能用吗？

    直接用 Claude 模型需要代理或中转。换国产模型则没有网络问题。
