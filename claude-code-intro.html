🎯 一句话理解：你告诉它"帮我实现用户登录功能"， 它不会只给你写出一个函数。它会读你的项目结构、制定方案、 跨文件写代码、跑测试、失败后修复、再跑，直到全部通过。
🤖 它跟普通 AI 编程工具有什么不同？

这是一个本质区别，不是程度的差异。

传统 AI 工具 (Copilot)
像自动补全——你写了一半代码，它猜你下一句。

工作范围局限在当前文件。你写一行它补一行，被动等待你操作。 可以做的事情只有补全代码。
Claude Code
像一个外包程序员——你给需求，它独立完成整个任务。

扫描整个项目、理解架构、规划方案、跨文件修改、 跑命令、查数据库、写测试、部署上线。全部自主完成。

⚙️ 核心原理：Agent 工作循环

Claude Code 不是生成一段文字就结束，而是一个持续自主循环：

1
理解需求
分析你的自然语言描述，明确目标
↓
2
读代码库
扫描项目结构，追踪文件依赖关系
↓
3
制定方案
决定改哪些文件，设计实现方式
↓
4
执行
编辑代码、运行 shell 命令、安装依赖
↓
5
验证
跑测试、检查 lint，确认结果
↓
6
迭代
测试失败→返回第4步修复，直到全部通过
▼ 完成

这叫"工具循环"。每次它决定用什么工具、 读什么文件、跑什么命令，结果反馈回来再做下一步决策。 这和人类写代码的过程完全一致：写→检查→发现不对→改→再检查。

🔧 进阶定制
📄
CLAUDE.md — 项目级"记忆文件"
项目根目录放一份，Claude Code 每次自动读。 写代码规范进去，生成的代码就自动符合你的风格。
🧩
Skills — 可复用的"技能包"
预封装的工作流程。比如"sql skill"让 Claude 分析数据库，"deploy skill"让 Claude 执行部署流程。
🔒
Hooks — 自动化"守门员"
强制执行的脚本。修改文件后自动格式化、 执行危险操作前弹确认——CLAUDE.md 是建议，Hooks 是必须。
🔗
MCP — 连接外部世界
连接数据库、API、Slack、Jira 等外部服务。 一个协议让 Claude 操作任何支持 MCP 的工具。
👥
Sub-agents — 多 Agent 并行
复杂任务拆给多个子 Agent：A 写后端、 B 写前端、C 写测试，各自独立上下文，并行干活。

⚠️ 它不是用来做什么的

不是设计工具（不会画 UI）· 不是产品经理（不会帮你定义需求） · 不能代替你思考（架构决策还得你来定）

本质是给程序员用的效率工具。虽然能写很多代码， 但你需要能读懂、评估、修正它的输出。

⏱ 最佳实践来自 Anthropic 内部专家 Boris Cherny（Claude Code 核心开发者）

❓ 常见问题
我不会编程，能用吗？
不推荐。虽然它能写很多代码， 但你需要能读懂和修正它的输出。
跟 Cursor / Copilot 比呢？
Claude Code 是 Agent，自己能跑完整个流程。 Cursor 的 IDE 体验更流畅，适合边写边改。
安全吗？贵吗？
每次执行命令都经过权限审批。Pro $20/月够个人用， 或 API 按量付费。换国产模型成本更低。
国内网络能用吗？
直接用 Claude 模型需要代理或中转。 换国产模型（通义、Kimi 等）则无网络问题。
它能做什么规模的代码库？
支持 200K token 上下文，可处理大型项目。 配合 Plan 模式，即使复杂重构也能先出方案再动手。
