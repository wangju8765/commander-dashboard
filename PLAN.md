## 一、项目背景

用户（Zeno）需要管理的事务维度多且杂：
- **项目**：多个并行推进的开发/运营项目
- **长期目标**：如数学教育产品矩阵，周期数月到年
- **短期目标**：如月度冲刺，周期数周
- **周计划**：本周日程、明天安排、待办清单
- **未来规划**：按月/季的时间轴安排
- **思考库**：务虚——读书笔记、理论反思、灵感
- **行动库**：务实——具体任务、实践记录

目前全靠脑力追踪，消耗大量认知能量。需要一个可视化的管理面板。

## 四、技术方案

- **语言**：纯 HTML + CSS + JS
- **存储**：浏览器 localStorage + JSON 导出/导入
- **部署**：本地打开即可用；后续可上 GitHub Pages
- **扩展性**：数据层接口化，方便后续加后端
